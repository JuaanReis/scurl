Referência da API

Base URL para todas as requisições: https://scurl.vercel.app

Autenticação & Limites

A API é pública e não requer tokens de acesso. O controle é feito via Rate Limiting por endereço IP.

Recurso Configuração / Resposta
Rate Limit Configurável via config["rate_limit"]. Retorna 429 Too Many Requests se excedido.
Concorrência Máximo de 20 scans simultâneos. Retorna 503 Service Unavailable se o limite for atingido sem enfileiramento.

Endpoints Disponíveis

POST /analyze

Executa um scan heurístico completo em uma URL.

Request Body (JSON)

CampoTipoObrigatórioDescrição
urlHttpUrlSimURL alvo. Validada pelo Pydantic.
use_cacheboolNãoSe true, busca resultado anterior no SQLite antes de escanear. Default: false.
POST /analyze
Content-Type: application/json

{
  "url": "https://example.com",
  "use_cache": false
}

Response 200 OK

{
  "scan": { ... },
  "target": { ... }
}

Erros

StatusCondição
400Engine retornou status: "error"
422URL inválida (Pydantic)
429Rate limit excedido
50320 scans já em execução

GET /scans/{identifier}

Retorna um scan completo por scan_id (UUID hex) ou url_hash (SHA-256 hex).

Path Parameter

ParâmetroTipoDescrição
identifierstrscan_id (UUID hex) ou url_hash (SHA-256 hex)

Erros

StatusCondição

Schemas de Resposta

ScanResponse

Objeto principal retornado após uma análise bem-sucedida.

{
  "status": "ok",
  "engine": {
    "name": "scurl",
    "version": "1.0.9"
  },
  "meta": {
    "scan_id": "13b0b10d192544f88b29755f7d9fbf16",
    "scan_time_s": 0.911,
    "url_hash": "e1d5e2c7...",
    "url": "https://youtube.com/",
    "threads": 2,
    "timestamp": "2026-05-19T03:01:58.423324+00:00"
  },
  "result": {
    "score": 7.8,
    "risk_level": "very low",
    "verdict": "safe"
  },
  "stats": {
    "rules_total": 36,
    "rules_triggered": 3,
    "trigger_rate": 0.083
  },
  "heuristics": [ ... ],
  "insight": []
}

Escala de Risco (result.risk_level)

ScoreNível de Risco
0 – 20very low
21 – 40low
41 – 60medium
61 – 80high
81 – 100very high

heuristics[]

Cada item representa uma regra que produziu resultado. Regras que retornam None não aparecem na lista.

{
  "name": "ssl_score",
  "category": "server",
  "value": 0.0,
  "weight": 3.5,
  "contribution": 0.0,
  "details": { ... },
  "reasons": ["..."]
}
CampoTipoDescrição
namestrIdentificador da regra
categorystrserver, html, url, domain
valuefloatScore bruto da regra (0.0–1.0)
weightfloatPeso da regra no pipeline
contributionfloatContribuição ponderada no score final
detailsdictDados coletados pela regra (varia por regra)
reasonslist[str]Explicações geradas pela regra

Códigos de Erro

StatusSignificado
400Falha interna na engine de análise (status: "error")
422URL malformada ou inválida (Pydantic)
429Rate limit excedido
50320 scans simultâneos em execução
404ID ou Hash não encontrado no banco de dados

ErrorResponse

Retornado com 400 quando o engine falha internamente.

{
  "status": "error",
  "meta": {
    "url": "https://example.com",
    "scan_time_s": 0.12,
    "version": "1.0.9"
  },
  "error": {
    "type": "connection_error",
    "message": "..."
  }
}
